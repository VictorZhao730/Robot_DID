1 Introduction
In this hands‑on, we will demonstrate how to run our own private Ethereum network based on the Proof of
Stake consensus (UZH Ethereum PoS), stake some UZHETH, and earn staking rewards.
Figure 1: UZH Ethereum PoS network and the relation between various components we will implement in these hands‑on.
Learning objectives for these hands‑on:
• Setup the execution node for the UZH Ethereum PoS network, using Rust Ethereum (Rust) client.
• Setup the consensus node for the UZH Ethereum PoS network, using Lighthouse client.
• Configure our existing Metamask wallet in web browser to connect to one of the public UZH Ethereum
PoS nodes.
• Bridge Ether from UZH Ethereum PoW network to UZH Ethereum PoS network.
• (Optional) Configuring and running validators on the UZH Ethereum PoS network and receiving rewards
to our Metamask wallet.
• (Optional) Gracefully exiting validators from UZH Ethereum PoS network.

Navigate to the vm_pos folder,
cd vm_pos

7.1 Validator Exit
WARNING: If you do not complete this step, your assignment is at risk of being penalized. Please make
sure to pay extra attention to this particular part of the hands‑on!
For Validator Exit, you don’t have to attempt this in the class room, as you will need to run your validators for
some time, possibly several hours after they have been activated. This is because you need to stake and earn
some ether rewards in order to attempt the assignment.
However, eventually, you do need to exit your validators from the network.
To do that, you have to use the ethdo module provided along with the other executables (be sure to run the
export command first, as shown in section 5)
Make sure your reth and lighthouse clients are fully synced, before running the following command!
ethdo validator exit --mnemonic="YOUR_VALIDATOR_MNEMONIC" --validator="VALIDATOR_PUBLIC_KEY"
--connection "http://YOUR_PUBLIC_IP:8005"
This step should explain why it is absolutely vital to keep your validator mnemonic, generated earlier, safe and
preserved somewhere. After you run the above command, your validator will start the proceedings to exit the
chain
Remember, if you have multiple validators, you need to run this command several times, as each command is
targeted at ONE validator (whose public key you are providing).
